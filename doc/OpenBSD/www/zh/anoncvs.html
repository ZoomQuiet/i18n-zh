什么是匿名 CVS?

匿名CVS是不管当前OpenBSD源码如何变化，而保持你的本地副本为最新状态的一种方法。除了可以保证开发进度的同步，还可以 跟 踪发行版本勘误表的patch。

匿名CVS与其它源码更新技术相比，最大的优势是直接与中心源码库或者镜像连接工作。这意味着你拥有整套的CVS命令来控制其 它 人做的源码修改与你的修改的合并与更新，以及执行差别比较,修改历史纪录和其它针对中心的查询。

OpenBSD项目当前有5个主要的源码库:

  • src - 存放OpenBSD操作系统的所有源码。
  • ports - 存放 OpenBSDPorts.
  • www - 存放所有的OpenBSDweb页(包括现在这个)。
  • X11 - 存放 OpenBSD对 XFree86-3 软件计划作修改后的版本。
  • XF4 - 存放 OpenBSD对 XFree86-4 软件计划作修改后的版本。

总而言之，匿名CVS真正的强大之处在于它是一个"宽容的" 源码控制系统 - 它尊重你对本地源码的修改并且 让 你以"最好的效果"来更新整个源码树，而不是往下进行之前必须解决一堆不可思议的问题。

什么是CVS?

CVS是用于管理 OpenBSD 源码树的源码控制系统。 它对所有正式发行的源码和修改实现一个中心 的库，却允许开发者维护他们本地的源码拷贝和修改。具有"write access"的开发者能直接将修改提交到OpenBSD源码树，而 "Anonymous CVS"用户具有"read access"可以让他们本地的源 码拷贝保持最新，并向中心库发送查询请求。

CVS 主要的强大之处是能对中心库的修改与本地拷贝的修改执行智能化合并。 就是说,如果你对某个模块作了修改并执行了更新 操 作，你的修改不会"blown away",被风吹走，而且CVS可以对两地的修改作最佳的合并。

如果出现修改没有完全合并完成，CVS提供了"soft fallback",它为你的本地拷贝提供带注解的修改，为你的版本保留一个未修改 拷贝并继续更新你所请求的其它源码模块。

开始使用匿名CVS

最新的CVS版本可以Cyclic获取。低于1.6的版本是不推荐使用，而且可能不能工作。 如 果你有一个安装好的OpenBSD,那么CVS 已经被包含在里面了。

拥有OpenBSD CD的用户也许能看到上面有CVS/ 目录。事实上原因是，该CD有一个OpenBSD src模块的checkout可 用 于更新。使用这个树来初始化CVS更新将比全新更新整个OpenBSD源码树快很多。有两种途径使用这个CD：

  • 将CVS树从CD拷贝至/usr/src(假设CD被mount在/mnt): 
        # cd /mnt; pax -rw CVS Makefile [a-z]* /usr/src
  • 与CD使用一个union mount(见 mount_union(8)) ，mount 到一个可写的目录。(只是在需要从该树编译的情况下使用。) 
        # mount -t union -o -b /mnt /usr/src

对于没有CD的用户，你可以使用 cvs(1) 来 "checkout" 源码库。这将在 下节讨论。

这之后，/usr/src 将成为checkout目录，在这里所有的 cvs(1) 命令都能正常工作了。

使用CVS来获取和更新源码树

CVS 被设计为一种简易的方法来检索和更新你的源码，因此你要做的相关事情不多。首先，你必须确定你要跟踪同步的是 current还是patch 分支。 current树反映所有最新的修改，而patch branch包含正式的发行版本以及errata发布的可 用patch。如果想了解关于current,请参考 Upgrade Mini-FAQ。

一旦你确定了你要跟踪同步的树，你必须选择要使用的匿名 CVS 服务器。下面是一个可用的服务器列 表 。有三种方式可以访问这些服务器。

ssh
Secure Shell可用来访问匿名CVS服务器。这是我们推荐使用的方式，因为它是加密的。从2.6起，OpenBSD已经把 OpenSSH包含在它的标准的发行版本内了。
rsh
Remote Shell对那些无法使用ssh的用户，可用于访问其中一些服务器 。
pserver
pserver对那些处于防火墙后面，被阻塞了其它两种连接方式的用户来说，很有用。

说明: 对想使用rsh的用户，你必须首先设置CVS_RSH 变量为rsh。

  • 对于Korn/Bourne shells: 
        $ export CVS_RSH=/usr/bin/rsh
  • 对于csh/tcsh: 
        % setenv CVS_RSH /usr/bin/rsh

一旦你选择好了你将使用的匿名CVS Server 以及使用的方法，你就可以开始使用cvs了。如果你有我们 的OpenBSD光盘，你可以使用光盘上面的CVS checkout ，按照上面的指导来获取源码到你的系统。如果你手上没有光盘，那么使用下面的方法来 checkout源码。这些方法把 OpenBSD 源码树放置在/usr/src 下面。

    # cd /usr; cvs checkout -P src

上面的命令将checkout current源码树。很多人将只想要patch分支的源码。如果要checkout patch分支，你必须在你的 命 令里添加标签。比如:

    # cd /usr; cvs checkout -P -rOPENBSD_3_5 src

对于3.4相应标签为OPENBSD_3_4 以此类推。

OPENBSD_3_5标签包含了正式发行版源码和已经发表可用的errata。

cvs(1)服务器使用范例

说明: 如果你想更新一个分支(比如一个patch branch) 到current，你要添加 -A 标记到cvs，但是该标记 在其它方面用处不大。一些老的 OpenBSD文档在很多范例中推荐使用这个标记。我们不再推荐使用这个标记，除非确实有必要。

一个使用匿 CVS服务器的范例将象下面这样：

% setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
% cd /tmp
% cvs get src/sys/arch/sparc
    [copies the files from the 库 to your machine]
% cvs log src/sys/arch/sparc/sparc/locore.s
    [shows the commit log for the chosen file]
% cvs diff -bc -r1.1 -r1.5 src/sys/arch/sparc/sparc/locore.s
    [shows the changes between revisions 1.1 and rev 1.5]

为了使用 ``pserver'' (一种直接的TCP连接，不使用ssh或者rsh)你必须登录一次:

% setenv CVSROOT :pserver:anoncvs@anoncvs.ca.openbsd.org:/cvs
% cvs login
(Logging in to anoncvs@anoncvs1.ca.openbsd.org)
CVS password: anoncvs
    [This writes a line to ~/.cvspass (filename over-ridden by CVS_PASSFILE).]
    [An example line from my ~/.cvspass after typing 'blah' for the above    ]
    [password is:                                                            ]
    [:pserver:anoncvs@anoncvs5.usa.openbsd.org:/cvs Au'yc                    ]
    [After logging in ONCE every other use of the above CVSROOT will work.   ]
% cvs get ksrc-i386 ksrc-common
    [Allows you to retrieve ONLY that necessary to rebuild an i386 kernel.   ]

这里是一个常规更新源码树的一个例子：

  • 首先`get'一个初始的源码树：

    (如果你要跟踪同步current):

        # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
    # cd /usr
    # cvs -q get -P src

    (如果你要跟踪同步 3.5 patch branch):

        # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
    # cd /usr
    # cvs -q get -rOPENBSD_3_5 -P src
  • 以后，你就可以 `update' 这个树了:

    (如果你要跟踪同步current):

        # cd /usr/src
    # cvs -q up -Pd

    (如果你要跟踪同步 3.5 patch branch):

        # cd /usr/src
    # cvs -q up -rOPENBSD_3_5 -Pd
    每次你运行这个命令就将同步你的/usr/src树。它不会破坏你的任何本地的修改，而是试图去合并修改。如果你使用obj目录(不 是 obj 符号链接) 你也许希望附加 "-I obj" 到 cvs 命令行。这将防止出现一个警告，大致是all the obj directories it is going to encounter which are not in the 库.
  • 说明: 如果你更新的源码树最初是取自别的服务器或者从光盘获取的，你必须添加-d $CVSROOT 选项到cvs。 
        # cd /usr/src
    # cvs -d $CVSROOT -q up -Pd

使用ports,与使用src类似:

  • (如果你要跟踪同步current):

        # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
    # cd /usr
    # cvs -q get -P ports

    (如果你要跟踪同步3.5的patch branch):

        # setenv CVSROOT anoncvs@anoncvs.ca.openbsd.org:/cvs
    # cd /usr
    # cvs -q get -rOPENBSD_3_5 -P ports
  • 然后，可以`update'这个树:

    (如果你要跟踪同步current):

        # cd /usr
    # cvs -q up -Pd ports

    (如果你要跟踪同步3.5的 patch):

        # cd /usr
    # cvs -q up -rOPENBSD_3_5 ports
在上面的例子中，-q 是可选的。只在想最少化cvs的输出时使用。如果希望看到完整的屏幕输出，那就可以不用。

想要与本地patched module作差异比较 (这里比如cd.c)，并写入到一个bug report:

    # cd /usr
    # cvs diff -u src/sys/scsi/cd.c > /tmp/patch

cvs(1) man page (包括在 CVS 源码中) 有许多关于如何使用 CVS 的信息。

X11 源码树

匿名CV服务器镜像也包括了OpenBSD X11 (XFree86)源码树。你可以对上面的指导略作修改，去更新你的取自第三方CD的XFree86 源 码树。 你可以拷贝或者下载XFree86 3 源码树，与X11 cvs模块一起。尽管current模块包含的XFree86 4就是XF4，还是有两种方法去获取 XF4 源码到/usr/XF4:

  • 从光盘拷贝树(假设第三方光盘mount在/mnt): 
            # cd /mnt; cp -Rp XF4 /usr
  • 用光盘，使用一个union mount，到一个可写的目录下。 
            # mount -t union -o -b /mnt/XF4 /usr/XF4
然后，/usr/XF4 将可以被cvs使用了。比如，你可以更新到-current源码树(假设你已经设置好了环境变量 CVSROOT): 
        # cd /usr/XF4
        # cvs -q update -Pd

警告: 当使用cvs的时候，你应该当心，当前的目录要么是树的根目录，或者因该位于一个单独的位置，比如 /tmp。一些命令，如"get"会在当前目录下创建一个任意的sub-tree，以后的更新将会递归的flesh out这个子树。

匿名CVS服务会给初学使用的开发者一个学习的机会来学习CVS的运作，在作"commit"提交前彻底熟悉这个系统 -- 在对开发者显 示 了它的一些有用的技巧和高品质之后，它自然而然的把访问的权利交给了开发者。最好还是，提供补丁的人们能创建与CVS树相关的 "diff"s，这样能够简化整合。

使用 rsh(1) 还是 ssh(1)?

缺省情况下，client使用ssh ("secure shell": OpenSSH)与CVS server交谈。

很多的CVS站点出于安全的原因不再支持rsh。本地的问题，像防火墙或者有缺陷的协议仿真器如slirp也可能阻碍rsh的使用。然 而 ，如果希望使用rsh,必须设置CVS_RSH这个环境变量指向rsh (通常/usr/bin/rsh)。

如果本地策略阻止外出的连接使用ssh的默认端口22，那么也可以使用2022。 要注意的是，不是所有的匿名CVS服务器都接受2022 口的ssh接入。此外，大多匿名CVS服务器不再接受none cipher, 因为出于安全的原因最近的版本禁止了这种无密 文的连接。还有，别开启压缩功能： CVS已经启动了压缩。

你可以在$HOME/.ssh/config配置文件里作下指定以避免上面提到的缺陷和限制：

    Host anoncvs.ca.openbsd.org
        Compression no
        Port 2022

CVS is a little noisy starting up; to quiet it a bit you may want to do this:

    % setenv CVS_CLIENT_PORT -1

使用sup(1)镜像CVS 库

想镜像OpenBSD CVS树本身的用户，现在可以镜像anoncvs.usa.openbsd.org或者anoncvs3.usa.openbsd.org (两个不同的服务器)。注意这是cvs 树，不是checked out 的源码树。这种镜像只是在你想更快的执行cvs操作 (diff, annotate, 等等) 或者你有多个源码树， 你只想传送一次数据的情况下才有用。(你可以从本地的cvs镜像checkout一个树)。

一个示范的supfile可能是这样的：

cvs host=anoncvs.usa.openbsd.org hostbase=/ base=/home delete

这将镜像cvs树到/home/cvs ，sup数据文件结束于/home/sup。整个OpenBSD cvs树当前大概2.2GB大小。

设置一个匿名CVS镜像

如果你想设置一个新的匿名CVS镜像站点，并让公众访问，请联系维护者。匿名CVS镜像 需要大概2.2GB磁盘空间。每个匿名CVS服务器用户好用32MB交换分区(假设用户执行大型的操作；而小型的操作使用更少的资源，匿 名 CVS仍然比ftp或者sup对系统的影响要小)。这样的匿名CVS服务器在所服务的区域应该拥有出色的网络连接性能。可以从这里得到一 份 文件,该文件描述了如何设立匿名的CVS服务器.

最后说明

在升级你的源码树后,你应该在build前阅读/usr/src/Makefile的开头部分。同样，如 果可能的话你应该在 make build之前build一个新的内核。某些情况下在你能build内核前，rebuild和安装config工具。如果 config GENERIC 失败，你就需要这样。

很重要的一点是，通过rebuild源码树以从一个release版本升级到current树会相当困难，因为依赖关系总是不明显，因此，建议 你尝试从源码build前安装最新的snapshot。
